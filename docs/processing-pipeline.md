### Validation Rules

| Check | Method | Action on Fail |
|-------|--------|----------------|
| Filename Sanitization | Regex `../` removal | Reject |
| Magic Bytes | Python `magic` | Reject |
| MIME Type | Content analysis | Reject |
| Extension | Whitelist | Reject |
| Size | Config limit | Reject |
| Malware | ClamAV (optional) | Quarantine |
| Archive Limits | Recursive scan | Reject |

### Sorting Rules Specification

#### Rule Types

| Rule Type | Description | Example |
|-----------|-------------|---------|
| **Static** | Extension-based | `.pdf` â†’ `/documents` |
| **Metadata** | Field-based | `author=john` â†’ `/john` |
| **Content** | Text-based | `invoice` in content â†’ `/invoices` |
| **AI** | ML-based | `classification=invoice` â†’ `/invoices` |
| **Scheduled** | Cron-based | `0 0 * * *` â†’ Archive old files |

#### Rule Syntax

```yaml
# Rule Definition Example
rules:
  - id: rule-001
    name: "Invoice Sorting"
    description: "Sort invoices by month and amount"
    priority: 100
    enabled: true
    
    conditions:
      - type: "extension"
        value: ".pdf"
      - type: "content_match"
        pattern: "invoice|invoice #|amount"
        case_sensitive: false
    
    actions:
      - type: "move"
        destination: "/sorted/invoices/{year}/{month}"
      - type: "rename"
        pattern: "invoice_{date}_{amount}.pdf"
      - type: "tag"
        tags: ["invoice", "finance"]
    
    scheduling:
      trigger: "on_upload"
      cron: null
```

#### AI Classification Rules

```yaml
# AI Classification Rule
- id: ai-classify-001
  name: "AI Document Classification"
  description: "Use ML to classify documents"
  priority: 50
  enabled: true
  
  conditions:
    - type: "always"
  
  actions:
    - type: "ai_classify"
      model: "document-classifier-v2"
      confidence_threshold: 0.8
      fallback_category: "uncategorized"
    
    - type: "move"
      destination: "/sorted/{classification}/{year}"
```

### Rule Processing Order

1. **Priority-based**: Higher priority rules execute first
2. **Short-circuit**: Stop on first matching rule (configurable)
3. **Cumulative**: Apply multiple matching rules (configurable)

## ğŸ”„ Workflow Automation

### Workflow Triggers

| Trigger Type | Description | Example |
|--------------|-------------|---------|
| `on_upload` | Triggered on file upload | Auto-sort new files |
| `on_schedule` | Cron-based execution | Nightly archive |
| `on_event` | Event-based trigger | On folder change |
| `on_manual` | Manual trigger | User-initiated batch |

### Workflow Example

```yaml
# Workflow: Invoice Processing Pipeline
workflow:
  id: "invoice-pipeline"
  name: "Invoice Processing"
  description: "Automated invoice sorting and approval"
  
  triggers:
    - type: "on_upload"
      conditions:
        - extension: ".pdf"
        - content_match: "invoice"
  
  stages:
    - name: "Extract"
      action: "extract_metadata"
      on_failure: "retry"
    
    - name: "Classify"
      action: "ai_classify"
      model: "invoice-classifier"
      
    - name: "Route"
      action: "route_by_amount"
      conditions:
        - amount < 1000: "auto_approve"
        - amount >= 1000: "manager_approval"
    
    - name: "Archive"
      action: "move_to_archive"
      
  notifications:
    - on_complete: "user"
    - on_approval_required: "manager"
```
